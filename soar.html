Platform features

The SOC your analyst would build

Built from real SOC workflows — not vendor slide-decks. Every capability is a first-class module you can enable, tune, audit, or disable. No forced autonomy, no opaque verdicts, no charge-per-alert.

3-layer IOC enrichment

Admin-authored rules, normalization-level hints, and regex IOC sweep — in that order. Tracker lists filter out internal IPs and known-benign indicators before any TI call is made. Cross-alert dedup via the observable store.

  • AbuseIPDB, AlienVault OTX, VirusTotal, MISP
  • Observable store with TTL-backed cache
  • Tracker lists — allowlist, blocklist, internal CIDRs

Auto-action rules

11 deterministic actions — set_status, assign_to, escalate_to_tier, add_tag, add_comment, suppress, trigger_playbook, link_to_case, correlate_and_case, and more. Dry-run first, promote to live. Every rule fires within a conflict-detection envelope.

  • Dry-run mode with full audit trail
  • Condition DSL: equals, in, in_tracker, not_equals…
  • Correlation-driven auto-case creation

DAG playbook engine

Visual DAG builder with 6 node types: start, action, condition, approval, ai_agent, end. Static paths always execute; AI nodes degrade gracefully if the LLM is unavailable. 8 seeded playbooks cover phishing, malware, identity, DLP, brand-protection, EDR, and universal triage.

  • Static and dynamic paths in one DAG
  • Approval gates — 24h TTL, audit-logged
  • Per-device-type trigger conditions

Autonomous SOC agents

8 pre-configured agents across L1 triage, L2 investigation, threat intel, incident response, and management. Three autonomy tiers (direct LLM → scoped agent → full autonomy). Device-type-aware prompts for firewall, EDR, email, identity, DLP, threat-intel, SIEM, and rating providers. Multi-provider LLM — OpenAI, Anthropic, Ollama, local.

  • L2-depth triage — not Tier-1 fluff
  • 44 MCP tools · publishable as an open MCP server
  • Guardrails: max turns, blocked tools, timeout, cost cap

Glass-box reasoning

No black-box verdicts. Every agent decision shows the tool calls made, the evidence gathered, the MITRE tactics matched, and the confidence score — displayed inline on the alert. If the verdict is wrong, the path to wrong is visible. Analysts coach the agent in natural language; the agent learns from every override.

  • Per-decision evidence chain with citations
  • MITRE ATT&CK tactic + confidence tagging
  • Natural-language coaching & override history

Normalization & routing

Webhook → Kafka → normalize → dedup. ECS-aligned field mapping handles poly-shape payloads (dict-or-list tolerance for nested fields). 9 pre-built normalization configs for Avanan, BitSight, CrowdStrike, Devo, Identity Protection, Microsoft DLP, SecurityScorecard, SOCRadar.

  • ECS & OCSF-aligned field names
  • Hash-based dedup with event-time precedence
  • Pipeline token guard — no out-of-band writes

Attack-chain intelligence

Cross-alert correlation turns isolated tickets into campaigns. Entity context, blast radius, related alerts, root-cause trace, MITRE phase prediction, attack-sequence scoring. Surfaces shared IOCs, timelines, and the adversary’s likely next move across your fleet.

  • Blast-radius quantification per alert
  • MITRE phase prediction on-demand
  • Shared-IOC campaign detection

Approval queue

AI agents pause before any dangerous tool call: block_ip, isolate_host, disable_user, delete_alert. The session goes to pending_approval, analyst reviews in the Approvals tab, approves or rejects, and the agent resumes with the result fed back in-context.

  • 24-hour TTL · auto-expire stale requests
  • Full context: tool, inputs, agent reasoning
  • Reject returns a message to the agent — no silent ignore

Case management

Every alert can escalate into a case or join an existing one via correlate-and-case rules. Cases bundle alerts, artifacts, comments, playbook runs, and AI sessions into a single investigation timeline with disposition and MTTR tracking.

  • Auto-linking by shared IOC + time window
  • Multi-alert disposition & merge
  • MTTR, MTTA, SLA breach KPIs

Self-learning knowledge hub

False-positive patterns surface automatically from resolved alerts and become auto-close rules for future matches. Runbooks, asset/user/network-zone lookups, and IOC history ground agent reasoning via the search_knowledge MCP tool. Your SOC gets smarter with every incident — not reset every time an analyst leaves.

  • Auto-FP pattern learning from resolutions
  • Runbook library — AI-groundable context
  • Asset, user, network-zone lookups

Agent governance & audit

Enterprise-grade AI governance, not an afterthought. Per-action-type autonomy toggles, one-click global kill switch, protected-network lists, model-version pinning, hash-chained audit trail of every agent decision. The controls compliance teams ask for — including ISO 42001-style attestations.

  • Per-action-type enable/disable · granular
  • Hash-chained immutable audit trail
  • Global kill switch · emergency stop

Threat intelligence — included

TI is a feature, not an upsell. AbuseIPDB, AlienVault OTX, VirusTotal, and MISP wired into the enrichment pipeline out of the box. Per-IOC caching and TTL-backed observable store keep provider calls low; provider health & usage tracked on every query. Zero extra line-item on the quote.

  • 4 TI providers wired — no extra integrations to buy
  • Cross-alert observable store · IOC dedup built-in
  • Provider health · rate limits · usage tracked

Architecture
