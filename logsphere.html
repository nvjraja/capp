Platform features

Everything you need — nothing you don’t

Engineered for SOC teams that care about both the bill and the audit.

OCSF 1.8.0 normalization

Every vendor log is normalized to an open schema shared with the industry. 13 target OCSF classes cover authentication, process, network, file, DNS, web, API, registry, account change, and more.

  • Open schema — not proprietary
  • Consistent fields across vendors
  • Three representations preserved per log

Per-source pipeline isolation

Every source runs its own worker, buffer, and Kafka consumer group. A noisy or slow feed cannot starve the rest. Config updates are lock-protected across batch processing.

  • Independent backpressure per source
  • Step-robust shutdown guarantees final flush
  • No noisy-neighbor starvation

Compliance-first storage

All logs are written to Parquet before filtering or dedup runs. Atomic writes (tmp → fsync → rename), a disk-space guard, and Kafka offsets committed only after fsync — no silent data loss.

  • Raw-before-reduce invariant enforced
  • Atomic, crash-safe Parquet writes
  • Partitioned by date for easy retention

Rules engine

Configurable field-match rules drive DROP, TAG, ROUTE, and TRANSFORM actions. Operators cover equals, contains, regex, ranges, and list membership. Per-log cycle tracking stops infinite fires.

  • Primary lever for SIEM cost reduction
  • Full action set — drop, tag, route, transform
  • Rule-cycle protection per log

Deduplication & aggregation

Event-time windowed dedup and aggregation collapse retransmits and repeating telemetry using your log’s real timestamp — not wall clock — so replay and late arrivals behave correctly.

  • Configurable windows per source
  • O(expired) eviction — no GC stalls
  • Preserves security signal while cutting volume

Multi-format parser

JSON, XML, KV, CEF, syslog, plain text — all handled. Per-source preset library covers Windows Security, Sysmon, Palo Alto, Fortigate, CloudTrail, and more, with thread-safe format overrides.

  • Broad format coverage out of the box
  • Preset library for common vendors
  • Narrow exception handling — no silent swallows

KQL log search

Kibana-style query syntax backed by a hand-written parser. Field names are regex-validated, values are bound as parameters — injection-proof by construction.

  • AND / OR / NOT, wildcards, ranges, free-text
  • Schema & field-values APIs for autocomplete
  • Auto-detects event-time column per source

DuckDB lakehouse queries

Columnar scans straight over Parquet — no indexer bottleneck. Parameterized SQL everywhere, union_by_name for schema evolution, snappy compression. Sub-second investigation on typical queries.

  • No proprietary index to maintain
  • SQL-injection-proof by design
  • Transparent schema evolution

SIEM forwarding

Route the filtered stream to any downstream SIEM. CRUD over the API, secret-lock UX on credentials, per-destination enable/disable. You pay the downstream bill only for what matters.

  • Multiple destinations per source
  • Credentials never re-exposed after save
  • Cost-reduced volume — not raw firehose
